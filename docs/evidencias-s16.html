Pipeline CI/CD integral (test + Docker Hub + Render)

Cierre final del curso con una aplicacion Hola Mundo contenedorizada, pruebas unitarias y despliegue continuo. El workflow de GitHub Actions ejecuta instalacion de dependencias, pruebas, build/push de imagen y trigger de deploy en Render.

Checklist requerido (S16)

  • 1. Repositorio con app Hola Mundo: app en semana13-despliegue-produccion/app/server.js.
  • 2. Prueba unitaria: casos en server.test.js para / y /health.
  • 3. Dockerfile: imagen Node 22 con healthcheck en /health.
  • 4. GitHub Actions CI/CD: npm ci -> npm test -> Docker Hub -> Render.

Flujo CI/CD en push a main

  1. Install dependencies: npm ci en semana13-despliegue-produccion/app.
  2. Unit tests: npm test con node --test.
  3. Docker Hub: login, build y push de tags latest, sha, fecha.
  4. Render: trigger de deploy por API con RENDER_SERVICE_ID.

Validacion tecnica

  • Docker Desktop: detectado localmente (docker info) version 29.2.1.
  • Pruebas locales: 2/2 casos en PASS.
  • Render health: https://devops-e2.onrender.com/health responde HTTP 200.

Secrets necesarios

  • DOCKER_USERNAME · usuario Docker Hub.
  • DOCKER_PASSWORD · access token Docker Hub.
  • RENDER_API_KEY · API key de Render.
  • RENDER_SERVICE_ID · id del servicio en Render.
